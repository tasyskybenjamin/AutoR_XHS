# 小红书运营实验协议

## 概述

本协议定义了 AI agent 进行小红书内容实验的标准流程。

## 核心原则

1. **数据驱动**：所有决策基于爆款数据分析
2. **快速迭代**：每次实验聚焦一个变量
3. **可量化评估**：使用 6 维度评分体系
4. **保留最优**：失败的经验也要记录

## 实验循环

```
┌─────────────────────────────────────────────────────────┐
│  1. 分析爆款                                             │
│     - 读取 data/trending/ 下的爆款数据                    │
│     - 提取高互动内容的共同特征                            │
│     - 更新 data/trending/analysis.json                   │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  2. 生成内容                                             │
│     - 基于爆款规律生成多个内容候选                        │
│     - 每个候选独立设计切入点                              │
│     - 输出到 output/ 目录                                │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  3. 评分排序                                             │
│     - 对每个候选进行 6 维度评分                           │
│     - 选择评分最高的作为发布版本                          │
│     - 记录所有候选到 results.tsv                         │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  4. 用户审核                                             │
│     - 用户手动发布最优内容                                │
│     - 可对内容进行微调                                   │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  5. 追踪结果                                             │
│     - 记录发布后的互动数据                                │
│     - 更新实验记录                                        │
│     - 识别有效策略                                        │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  6. 迭代优化                                             │
│     - 根据数据调整生成策略                                │
│     - 开始下一轮实验                                      │
└─────────────────────────────────────────────────────────┘
```

## 评分体系

内容必须满足以下 6 个维度：

| 维度 | 评分标准 | 分值 |
|------|----------|------|
| 1. 开场钩子 | 第一句话有具体数字或强反差 | 0-1 |
| 2. 生活比喻 | 用日常生活比喻解释概念 | 0-1 |
| 3. 可验证数字 | 有具体可验证的数字（不是"效果很好"） | 0-1 |
| 4. 无套话 | 避免"改变游戏规则"等空话 | 0-1 |
| 5. 具体问句 | 结尾问句具体，用户可直接回答 | 0-1 |
| 6. 字数范围 | 字数在 150-600 字之间 | 0-1 |

**总分**：0-6 分，**≥4.5 分** 才建议发布

## 内容类型策略

### 爆款内容模式

1. **数字型**：具体数字 + 效果展示
   - "30天瘦了20斤，我做了这三件事"

2. **对比型**：前后对比 + 强反差
   - "从路人甲到焦点，我靠这3招"

3. **教程型**：步骤清晰 + 可操作
   - "5步教你画出高级感日常妆"

4. **测评型**：真实体验 + 中立客观
   - "用了3个月的贵妇霜，说说真实感受"

### 标题公式

```
公式1: [数字] + [效果] + [动作]
  例：7天淡纹，我找到了平价替代

公式2: [身份] + [反差] + [解决方案]
  例：作为一个油皮，我终于找到了不脱妆的秘诀

公式3: [疑问] + [答案]
  例：为什么你涂口红显老？问题出在这里
```

### 封面文案

- 10 字以内
- 突出核心卖点
- 使用强对比色
- 避免过度设计

## 标签策略

每次发布使用 3-5 个标签：

| 类型 | 示例 |
|------|------|
| 领域标签 | #美妆 #穿搭 #护肤 |
| 场景标签 | #上班族 #学生党 #新手 |
| 热点标签 | #春季 #开学季 #618 |
| 风格标签 | #日常妆容 #简约穿搭 |

## 实验记录

每次实验记录到 `data/experiments/results.tsv`：

```tsv
date	topic	title	score	viral_score	likes	collects	comments	verdict
2026-04-06	春季护肤	7天养出好皮肤	5.5	0.8	1234	567	89	keep
2026-04-05	穿搭分享	显瘦穿搭技巧	4.0	0.6	456	123	34	discard
```

## 注意事项

1. **不要一次发布太多内容**：保持账号活跃度但不要刷屏
2. **保持内容一致性**：账号定位要清晰
3. **关注数据反馈**：根据互动调整策略
4. **避免违规内容**：不要夸大效果，不要虚假宣传
5. **尊重原创**：可以模仿爆款形式，但要有自己的风格

## 成功标准

- 点赞率 > 5%（点赞/曝光）
- 收藏率 > 3%（收藏/曝光）
- 评论率 > 1%（评论/曝光）
- 关注转化率 > 0.5%

如果连续 3 次实验得分 < 4 分，重新分析爆款数据，调整策略。
